## 一、需求概述

在首页轮播 Banner 下方，新增「品类楼层」区域，按指定的 6 个一级品类分楼层展示商品，参考鑫方盛首页的品类商品展示形式。

**目标用户**: 东非买方（未登录/已登录均可见，首页为公开页面）

## 三、6 个楼层品类

| 序号 | 楼层名称 | 说明 |
|------|---------|------|
| 1 | 工具耗材 | Tools & Consumables |
| 2 | 劳保安防 | Safety & Protection |
| 3 | 紧固密封 | Fasteners & Sealing |
| 4 | 工控配电 | Industrial Electrical |
| 5 | 门窗五金 | Doors, Windows & Hardware |
| 6 | 装饰建材 | Decoration & Building Materials |

**i18n**: 楼层名称需三语（zh/en/sw），硬编码在前端翻译文件 `messages/*.json` 中。

**品类映射**: 前端维护 `FLOOR_CATEGORIES` 常量数组，每项包含:
- `code`: 对应 DB 中 L1 品类的 `category_code`（如 `"01"`）
- `nameKey`: i18n 翻译 key（如 `"floorToolsConsumables"`）
- `themeColor`: 楼层主题色（渐变色值）
- `bgImage`: 背景图 URL

当 DB 中尚无对应品类或该品类下无商品时，楼层正常渲染但商品区显示"商品即将上架"占位。

## 五、楼层电梯（FloorElevator）

参考京东 PC 首页的「楼层电梯」交互。

### 5.1 行为

| 行为 | 说明 |
|------|------|
| 出现时机 | 滚动到品类楼层区域时出现，离开时隐藏 |
| 定位 | `position: sticky`，左侧，纵向居中 |
| 宽度 | ~80px |
| 内容 | 6 个品类短名 + "↑ 回顶部" 按钮 |
| 高亮 | IntersectionObserver 检测当前可见楼层，对应项高亮 |
| 点击 | `scrollIntoView({ behavior: 'smooth' })` 平滑滚动 |

### 5.2 视觉

## 六、精简商品卡（ProductCardCompact）

对比现有 `ProductCard`（商城列表页完整版），楼层用精简版：

| 元素 | 列表页 ProductCard | 楼层 ProductCardCompact |
|------|-------------------|------------------------|
| 图片 | aspect-square, p-2 | 正方形, 略小, p-1.5 |
| 商品名 | 2行 line-clamp-2 | **1行 line-clamp-1** |
| 描述 | 有（line-clamp-2） | **去掉** |
| 品类徽章 | 有 | **去掉**（已在楼层标题） |
| 认证徽章 | 有 | **去掉** |
| 供应模式标签 | 标题前缀 | **去掉** |
| MOQ | 有 | **保留（精简格式）** |
| 询价按钮 | 有 + 购物车按钮 | **仅保留询价按钮（小号）** |
| 购物车按钮 | 有 | **去掉** |
| 点击行为 | 跳转商品详情 | **跳转商品详情**（同） |

**尺寸**: 卡片宽度约 200-220px（随容器自适应），高度约 280px。

## 八、数据获取策略

### 8.1 品类树（L2 子分类）

复用现有 `useCategoryTree()` hook，从树中按 L1 code 过滤出子分类列表。

### 8.2 楼层商品

**方案**: 前端并发 6 次商品列表 API 请求。

```
GET /api/v1/products?category_code={L1_code}&featured=true&size=8&sort=newest
```

- 优先返回 `is_featured=true` 的商品，不足 8 个则用非精选补充
- 每个楼层独立 SWR 缓存，独立 loading 状态
- 无数据时渲染占位 UI

**注**: 当前 API 的 `featured` 参数是过滤而非排序优先。若需要"精选优先+时间排序"的混合逻辑，有两种方案：

| 方案 | 说明 | 工作量 |
|------|------|--------|
| A: 前端两次请求合并 | 先请 featured=true size=8，不足再请 size=剩余数 | 无后端改动，前端逻辑略复杂 |
| B: 后端新增排序参数 | `sort=featured_first` 让后端 ORDER BY is_featured DESC, created_at DESC | 后端改一行，前端简单 |

**推荐 B**，后端改动极小（service 层加一个排序分支）。

## 十、响应式适配

## 十一、不做的事项

| 内容 | 理由 |
|------|------|
| 采购排行榜 | 本期不做，数据量不足 |
| 楼层内"查看全部"链接 | 用户确认不需要 |
| 楼层内 Tab 切换子品类 | 过度设计，MVP 不做 |
| 商品卡片轮播 | 静态网格即可，交互更简单 |
| 楼层懒加载 | 6 个楼层数据量不大，一次加载 |
