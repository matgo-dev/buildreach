# 询价报价 12a · 报价行部分跳过（Partial Skip）v1

> 来源：工单 12 买方报价决策实现过程中，发现运营无法对"采不到"的行项做标记，当前全覆盖校验阻塞了真实业务场景。
> 本工单补充：允许运营在报价回填时跳过部分行项，买方可见跳过状态和原因。

---

## 背景与动机

原设计：`create_quote` 强制要求 `payload.lines` 覆盖全部未删 `rfq_items`，每行必须有 `unit_price`。

**问题场景**：买方一次询了 5 个 SKU，其中 1 个运营无法采购（缺货/断供/不在经营范围）。运营没有办法只报 4 行——要么填个假价格，要么让买方改询价单删那行。

**行业参考**：阿里巴巴国际站的供应商可以只报部分商品。但阿里是多供应商竞标模式，我们是平台统一报价，所以采用更轻量的方案——仍要求全行覆盖提交，但允许标记 `skipped`。

---

## 设计决策

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| 跳过方式 | A. 不传该行 / B. 传但标记 skipped | **B** | 全覆盖保证数据完整性，每行都有明确处理结果 |
| 跳过原因 | A. 必填 / B. 选填 / C. 不要 | **B** | MVP 不强制，但允许运营写 |
| 全部跳过 | A. 允许 / B. 拦截 | **B** | 全跳过 = 没报价，语义不成立 |
| 跳过行状态 | A. 新增 QuoteItemStatus 枚举 / B. 布尔字段 | **B** | MVP 最简，布尔够用 |

## 后端逻辑变更

### 请求校验（`services/quote.py`）

```
for line in payload.lines:
    ① rfq_item_id 必须属于本 RFQ（不变）
    ② 不能重复（不变）
    ③ 非 skipped 行必须有 unit_price → QuoteLineNoPriceError(40514)

全行覆盖校验保持不变 → QuoteLinesIncompleteError(40513)
全部 skipped 拦截 → QuoteLinesIncompleteError(40513)
```

### 金额计算

- `line_amount`：skipped 行 = `None`
- `total_amount`：只汇总非 skipped 行的 `line_amount`

### 行写入

skipped 行：`unit_price=None, moq=None, cbm=None, weight=None, line_amount=None, skipped=True, skip_reason=<用户输入>`

### 响应 DTO

`QuoteItemBuyerPublic` 和 `QuoteItemOperatorView` 均新增 `skipped: bool` + `skip_reason: str | null`。

## 前端变更

### 运营报价回填页（`operator/rfqs/[id]/quote/page.tsx`）

- 每行右侧新增 **skip 开关**（toggle switch，红色态）
- 打开后该行所有输入禁用变灰，商品名加删除线
- 可选填跳过原因（小输入框）
- 合计只算非 skipped 行
- 前端校验：不能全部跳过

### 买方详情页报价区块（`buyer/rfqs/[id]/page.tsx`）

- skipped 行：商品名灰色删除线 + "无法供应"灰色标签 + 原因（如有）
- 非 skipped 行展示不变

### 新增 i18n key

| key | zh | en |
|-----|----|----|
| `quote.skipLabel` | 跳过 | Skip |
| `quote.skippedLabel` | 无法供应 | Unavailable |
| `quote.skipReasonPlaceholder` | 原因（选填） | Reason (optional) |
| `quote.allSkippedError` | 不能跳过全部行项 | Cannot skip all line items |
| `error.quote.line_no_price` | 非跳过行必须填写单价 | Non-skipped quote line must have a unit price |
