# 央企专区 图片+分类回挂 · 交付规范(给数据提供方)

目的:每批交付**你们自己先校验**,过了再发。省掉我方逐批 debug、你方反复返工的循环。

## 交付目录结构

```
run_YYYYMMDD_HHMMSS/
  matches.json          ← 主数据,结构见 matches_schema.json
  images/
    <offer_id>/main_01.jpg, detail_01.jpg, ...
```

## 三条铁律(每批翻车都出在这)

1. **`matched_category_path` 必须是数组**,不是 `"A / B / C"` 字符串。
   我方类目名本身含 `/`(如 `铜/铝端子`、`DC/DC转换器`),字符串拆分必错。
   → 正确:`["土建材料","预应力","预应力精轧螺纹钢","预应力精轧螺纹钢"]`

2. **分类只能落到我方类目白名单里**(`类目白名单_叶子.csv`,6168 个叶子 / 32 个一级)。
   不要按 XFS 自家分类树打标。逐级名字要能在白名单里命中到叶子;
   命不中就 `matched_category_path: []` 并在 `note` 标"待定",**不要硬套**。
   更稳的做法:直接填 `matched_leaf_code`(白名单里的 code),就不用我方再解析路径。

3. **回挂靠 `material_name` 按名字匹配我方商品**,用词必须与我方材料表一致;
   同名不同规格我方只存一份图,**不要按规格拆成多条**(如"金刚石切割片(有豁口)/(无豁口)"应合成一条"金刚石切割片")。
   `material_category_code`(01–17)只在材料名重名时用来消歧。

图片:`main` 至少一张,最佳主图 min(宽,高) **≥ 600px**,否则该条判低清、不换图。

## 发货前自检

```
python validate_matches.py run_YYYYMMDD_HHMMSS/
```

- 打印:可落叶子数 / 不在白名单数 / 缺图 / 格式硬错。
- **有硬错(格式、缺字段、缺图)= 退出码非 0,别发货。**
- "不在白名单"的条目会列出来:尽量改映射到白名单;实在没有对应类目的,汇总给我方评估是否扩类目(不要自己在路径里造)。

## 文件清单

- `matches_schema.json` —— JSON Schema,字段与类型契约。
- `类目白名单_叶子.csv` —— 允许的落点(`leaf_code, category_path, l1`)。我方类目树有变更时会更新。
- `validate_matches.py` —— 自检脚本,与白名单同目录即可跑。
