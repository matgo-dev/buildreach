# 鑫方盛（xfs.com）采集数据规范

## 一、背景

我们从多个平台采集建材/工业品商品数据，导入自有 B2B 平台。此前已从阿里国际站（alibaba.com）采集了一批数据，现在需要从鑫方盛（xfs.com）采集新一批。

**目标**：拿到商品的基础信息、全部属性、全部图片，按统一格式存放，便于后续批量导入。

**商品详情页示例**：`https://www.xfs.com/productsku/10514704.html`

### 品类范围

**优先采集的 6 大类**：工具耗材、劳保安防、紧固密封、工控配电、门窗五金、装饰建材

具体哪些品类要爬、哪些不爬，**参见随文档一起发的导航截图标注**。

### 交付节奏（三步走）

**第一步：试采 2-3 个品类**
- 先挑 2-3 个品类（如"手动工具"、"紧固密封"），把品类树 + 该品类下的商品一起交付
- 我们验证数据格式和导入流程没问题后，再进入下一步

**第二步：交付全量品类树**
- 交付完整的 categories_raw.json，覆盖所有一级品类及其下子品类
- 我们确认最终采集范围

**第三步：批量爬商品**
- 按确认的范围批量采集

具体采集方式（翻页策略、并发、反爬处理等）由采集同事自行决定，我们只定义数据格式和存储规范。

## 二、输出目录结构

请严格按以下目录结构组织输出数据：

```
output_xfs_YYYYMMDD_HHMMSS/
├── run.json                        # 本次采集的元数据
├── categories_raw.json             # 鑫方盛品类树（如果能拿到）
├── categories/
│   └── {一级品类名}/
│       └── {二级品类名}/
│           └── ... (按实际层级)
│               └── offers/
│                   └── {商品编码}/          # 用鑫方盛的"商品编码"作为目录名
│                       ├── offer.json      # 商品结构化数据（下面有字段说明）
│                       └── images/         # 所有图片下载到本地
│                           ├── main_01.jpg # 商品主图/轮播图
│                           ├── main_02.jpg
│                           ├── ...
│                           ├── detail_01.jpg # 详情长图（商品介绍tab里的图片）
│                           ├── detail_02.jpg
│                           └── ...
```

**品类目录**：鑫方盛有清晰的品类面包屑（如 `动力工具 > 电动工具 > 电锤电镐工具 > 电锤工具`），按实际层级建目录。

**商品目录名**：用鑫方盛的"商品编码"（如 `10514704`），全局唯一。

## 四、categories_raw.json — 品类树（必须交付）

将采集到的所有品类**拍平成数组**，每行一个品类节点，靠 `parent_zh` 挂父子关系。

品类层级**以商品详情页面包屑为准，有几层记几层**（可能 3 级也可能 4 级，不固定）。导航栏只是辅助入口，不作为层级依据。

```json
[
  {"level": 1, "name_zh": "动力工具", "short_name": "动力", "path_zh": "动力工具", "parent_zh": null},
  {"level": 2, "name_zh": "电动工具", "path_zh": "动力工具/电动工具", "parent_zh": "动力工具"},
  {"level": 3, "name_zh": "电锤电镐工具", "path_zh": "动力工具/电动工具/电锤电镐工具", "parent_zh": "电动工具"},
  {"level": 4, "name_zh": "电锤工具", "path_zh": "动力工具/电动工具/电锤电镐工具/电锤工具", "parent_zh": "电锤电镐工具"},

  {"level": 1, "name_zh": "紧固密封", "short_name": "紧固", "path_zh": "紧固密封", "parent_zh": null},
  {"level": 2, "name_zh": "螺母", "path_zh": "紧固密封/螺母", "parent_zh": "紧固密封"},
  {"level": 3, "name_zh": "六角螺母", "path_zh": "紧固密封/螺母/六角螺母", "parent_zh": "螺母"},
  {"level": 3, "name_zh": "法兰螺母", "path_zh": "紧固密封/螺母/法兰螺母", "parent_zh": "螺母"}
]
```

| 字段 | 说明 | 必填 |
|------|------|------|
| `level` | 层级深度（1=一级品类，2=二级，以此类推） | **是** |
| `name_zh` | 品类中文名（面包屑全称） | **是** |
| `short_name` | 导航栏简称（如"动力"、"紧固"），**仅一级品类需要填** | 一级必填 |
| `path_zh` | 从一级到当前节点的完整路径，用 `/` 分隔（**唯一标识**） | **是** |
| `parent_zh` | 父品类的 `name_zh`，一级品类填 `null` | **是** |

**注意**：
- **所有品类名称以商品详情页面包屑里显示的为准**（如面包屑里是"动力工具"就用"动力工具"，不用导航栏的缩写"动力"）
- 有几层就记几层，不要自己合并层级
- `path_zh` 是品类的唯一标识，用于防止不同一级品类下出现同名子品类时的歧义
- 所有在 offer.json 的 `source_category_path` 中出现的品类名，都必须在这个文件里有对应节点

## 五、offer.json — 单商品数据（核心）

每个商品一个 `offer.json`，字段定义如下：

```jsonc
{
  // ========== 来源信息 ==========
  "source": {
    "platform": "xfs.com",            // 固定值
    "product_id": "10514704",          // 商品编码（页面上的"商品编码"字段）
    "product_url": "https://www.xfs.com/productsku/10514704.html",
    "crawled_at": "2026-06-22T10:00:00Z"   // ISO8601 UTC
  },

  // ========== 品类路径 ==========
  // 从面包屑提取，按层级排列
  "source_category_path": ["动力工具", "电动工具", "电锤电镐工具", "电锤工具"],

  // ========== 商品名称 ==========
  // 鑫方盛是纯中文站，只采集中文，英文由我们平台翻译API自动生成
  "product_name_zh": "东成 电锤FF28 28款不调速 FF28 960W 1000r/min 220V",

  // ========== 品牌 ==========
  "brand": "东成",

  // ========== 价格 ==========
  // 【本批次不采集】价格、优惠券、会员价等价格类信息暂不需要，后续按需补采
  // "price_tiers": [],

  // ========== 起订量 & 件装 ==========
  "moq": {
    "value": 1,
    "unit": "台",
    "original": "1台"        // 页面原文
  },
  "packing_qty": {
    "value": 3,
    "unit": "台/件",
    "original": "3台/件"     // "件装数量"字段原文
  },

  // ========== 结构化属性 ==========
  "attributes": [
    {
      "key": "型号",              // 属性名（中文）
      "value": "FF28",            // 属性值
      "group": "规格参数"         // 分组，见下方说明
    },
    {
      "key": "输出功率",
      "value": "960W",
      "group": "规格参数"
    },
    {
      "key": "空载转速",
      "value": "1000r/min",
      "group": "规格参数"
    },
    {
      "key": "电压",
      "value": "220V",
      "group": "规格参数"
    },
    {
      "key": "是否带正反转",
      "value": "否",
      "group": "规格参数"
    },
    {
      "key": "最大钻孔直径",
      "value": "28mm",
      "group": "规格参数"
    },
    {
      "key": "包装清单",
      "value": "电锤*1台",
      "group": "包装信息"
    }
  ],

  // ========== 配送 & 库存提示 ==========
  "delivery_info": {
    "estimated_shipping": "次日",  // 或 "1~3天"、"3~7天" 等，页面原文
    "original": "次日 / 1~3天"
  },

  // ========== 图片 ==========
  // gallery = 商品主图/轮播图（详情页顶部的图片）
  "gallery": [
    {
      "path": "images/main_01.jpg",           // 本地存放路径（相对于 offer.json）
      "source_url": "https://img.xfs.com/..."  // 原始图片URL
    }
  ],

  // description_images = 商品介绍tab里的详情长图
  "description_images": [
    {
      "order": 1,
      "path": "images/detail_01.jpg",
      "source_url": "https://img.xfs.com/..."
    },
    {
      "order": 2,
      "path": "images/detail_02.jpg",
      "source_url": "https://img.xfs.com/..."
    }
  ],

  // ========== 视频 ==========
  "video_url": null   // 如果商品有视频就填URL，没有就 null
}
```

## 六、字段采集来源对照表

以下是鑫方盛详情页上各个数据点 → offer.json 字段的映射：

| 页面位置 | 页面字段名 | offer.json 字段 | 必填 | 备注 |
|---------|-----------|----------------|------|------|
| 面包屑 | 品类路径 | `source_category_path` | **是** | 如 `动力工具 > 电动工具 > ...` |
| 商品标题 | 商品全名 | `product_name_zh` | **是** | 页面大标题 |
| 基本信息 | 品牌 | `brand` | **是** | |
| 基本信息 | 商品编码 | `source.product_id` | **是** | 如 10514704 |
| ~~基本信息~~ | ~~售价~~ | ~~`price_tiers`~~ | **跳过** | 本批次不采集价格类信息 |
| 基本信息 | 最少订量 | `moq` | **是** | |
| 基本信息 | 件装数量 | `packing_qty` | 否 | 有就填 |
| 基本信息 | 预计出货 | `delivery_info` | 否 | 原样存文本 |
| 规格参数tab | 所有 key:value | `attributes[]` | **是** | 逐条提取 |
| 规格参数tab | 包装清单 | `attributes[]` (group=包装信息) | 否 | |
| 商品图片区 | 轮播图 | `gallery[]` | **是** | 下载到 images/main_*.jpg |
| 商品介绍tab | 详情长图 | `description_images[]` | **是** | 下载到 images/detail_*.jpg |
| 详情页 | 视频 | `video_url` | 否 | 有就填 |
| URL | 商品链接 | `source.product_url` | **是** | |

## 七、图片下载规则

1. **全部下载到本地** — 不要只存 URL，图片必须下载到 `images/` 目录
2. **命名规则**：
   - 商品主图/轮播图：`main_01.jpg`, `main_02.jpg`, ... （按页面顺序）
   - 详情长图：`detail_01.jpg`, `detail_02.jpg`, ... （按页面从上到下顺序）
3. **保留原始格式** — jpg 就存 jpg，png 就存 png，不要转换
4. **下载最大尺寸** — 如果 URL 有尺寸参数（如 `_200x200.jpg`），去掉尺寸后缀拿原图
5. **source_url 必须保留** — offer.json 里同时记录原始 URL，方便后续校验

## 八、属性采集规则

### 分组（group）
| group 值 | 来源 |
|----------|------|
| `规格参数` | "规格参数"tab 里的技术参数（型号、功率、转速、电压等） |
| `基本信息` | 页面顶部的商品编码、件装数量、商品规格等 |
| `包装信息` | 包装清单 |

### 注意事项
- **每个 key-value 对独立一条**，不要把多个属性合并成一个字符串
- **value 保留原文**，不要自己换算或格式化（如 "1000r/min" 就存 "1000r/min"，不要拆成数值）
- 如果同一商品在不同位置出现了相同属性（如"品牌"在基本信息和规格参数都有），**只保留一条**，优先取规格参数 tab 的

---

## 九、本批次不采集的数据（明确跳过）

以下数据**本批次不需要采集**，后续如有需要会单独补采：

| 跳过项 | 说明 |
|--------|------|
| 售价 / 会员价 / 优惠券价 | 所有价格类信息都不采集 |
| 优惠活动信息 | 满减、折扣等促销信息 |
| 库存数量 | 不需要 |
| 配送费用 | 不需要 |
| 商品评价 | 评论数据不采集 |
| 服务保障 | "服务保障"tab 内容不采集 |

offer.json 中 `price_tiers` 字段可以**整个省略**或留空数组 `[]`。

## 十、与阿里国际站数据的差异

| 差异点 | 阿里国际站 | 鑫方盛 |
|--------|-----------|--------|
| 语言 | 中英双语 | **仅中文**（英文由我们平台翻译API生成，采集不用管） |
| 商品ID | offer_id (纯数字) | 商品编码 (纯数字) |
| SKU变体 | 有（颜色/规格切换） | **大部分无变体**，一个页面=一个SKU |
| 属性结构 | 有 selectable 标记 | 无，全部是展示型属性 |
| 价格 | 可能有阶梯价 | 通常单一价格 |
| 详情描述 | 有文字描述+图片 | **主要是纯图片**（长图） |
| 品类层级 | 不稳定 | 清晰的面包屑（以面包屑实际层级为准） |

## 十一、数据质量检查点

采集完后请自检以下项目：

- [ ] 每个商品都有 `product_name_zh`（不为空）
- [ ] 每个商品都有 `source_category_path`（至少2级）
- [ ] **品类信息三处一致**：每个商品的 `source_category_path` 中出现的每一级品类名，必须在 `categories_raw.json` 中有对应节点，同时与文件目录结构吻合。三处品类名不能有拼写差异、多余空格或层级错乱
- [ ] 每个商品至少有 1 张 `gallery` 主图且已下载
- [ ] `attributes` 至少有 2 个属性（型号、品牌通常都有）
- [ ] ~~`price_tiers` 至少有 1 条价格记录~~（本批次不采集价格）
- [ ] 图片文件实际存在且非空（>1KB）
- [ ] `source.product_url` 可正常访问
- [ ] 无重复商品（按 `product_id` 去重）

## 十二、交付物

1. **压缩包**：`output_xfs_YYYYMMDD_HHMMSS.zip`，包含完整目录结构
2. **数据量统计**：总商品数、各品类商品数、图片总数
3. **问题清单**：采集过程中遇到的异常（如某些商品无法访问、图片下载失败等）

如有任何字段存疑，**先按页面原文存储**，标注 `"_note": "待确认"` 即可，后续我们会在导入时处理。
